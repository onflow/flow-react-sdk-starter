# AGENTS.md

This file provides guidance to AI coding agents (Claude Code, Codex, Cursor, Copilot, and others)
when working in this repository. It is loaded into agent context automatically — keep it concise.

## Overview

Minimal Next.js starter template for building Flow blockchain applications with `@onflow/react-sdk`.
Uses the App Router, React 19, TypeScript, and Tailwind v4. Ships with four interactive tutorial
components (wallet connect, testnet faucet, balance query, FLOW transfer) wired to Cadence scripts
and transactions that are loaded as raw strings via a Turbopack loader rule.

Key pinned versions (from `package.json`): `next ^16.1.1`, `react ^19.2.0`, `@onflow/react-sdk ^0.11.0`,
`@onflow/fcl ^1.20.3`, `typescript ^5`, `tailwindcss ^4`.

## Build and Test Commands

- `npm install` — install dependencies
- `npm run dev` — start the Next.js dev server on `http://localhost:3000`
- `npm run build` — production build
- `npm run start` — serve the production build

No `lint` or `test` scripts are defined in `package.json`. Cadence tests under `cadence/tests/`
run via the Flow CLI (`flow test`).

### Flow Emulator workflow (optional)

1. `flow emulator start` — local Flow node on `http://localhost:8888`
2. `flow dev-wallet` — dev wallet on `http://localhost:8701`
3. Edit `src/components/flow-provider-wrapper.tsx` to swap the Testnet config for the Emulator
   block documented in `README.md`
4. `npm run dev`

## Architecture

- `src/app/` — Next.js App Router. `layout.tsx` wraps the tree in `FlowProviderWrapper`;
  `page.tsx` is a Server Component that renders `FlowHeader` + `FlowContent`.
- `src/components/flow-provider-wrapper.tsx` — client component that configures `FlowProvider`
  from `@onflow/react-sdk` with Testnet endpoints and passes `flowJson={flowJSON}` imported
  from the repo-root `flow.json`.
- `src/components/flow-header.tsx` — header with the `<Connect />` button and a network
  indicator driven by `useFlowConfig()`.
- `src/components/flow-content.tsx` — client component composing the four tutorial cards.
  The faucet card renders only when `flowNetwork === "testnet"`.
- `src/components/tutorial/wallet-connect.tsx` — step 1, uses `<Connect />`.
- `src/components/tutorial/faucet-funding.tsx` — step 2, links to `faucet.flow.com`.
- `src/components/tutorial/flow-balance.tsx` — step 3, uses `useFlowQuery` with
  `@cadence/scripts/GetFlowBalance.cdc`.
- `src/components/tutorial/send-flow.tsx` — step 4, uses `useFlowMutate` with
  `@cadence/transactions/TransferFlow.cdc`.
- `src/types/cadence.d.ts` — ambient module declaration for `*.cdc` imports.
- `cadence/contracts/Counter.cdc`, `cadence/scripts/{GetCounter,GetFlowBalance}.cdc`,
  `cadence/transactions/{IncrementCounter,TransferFlow}.cdc`,
  `cadence/tests/Counter_test.cdc`.
- `flow.json` — declares the local `Counter` contract plus dependencies (`FlowToken`,
  `FungibleToken`, `NonFungibleToken`, `MetadataViews`, `ViewResolver`, `Burner`,
  `FungibleTokenMetadataViews`) with emulator/testnet/mainnet aliases.

## Conventions and Gotchas

- **Cadence files are imported as strings.** `next.config.ts` registers a Turbopack rule that
  runs `*.cdc` through `raw-loader`. New `.cdc` files are consumable via
  `import X from "@cadence/..."` or a relative path.
- **Path aliases** (from `tsconfig.json`): `@/*` → `src/*`, `@cadence/*` → `cadence/*`.
- **Client vs Server Components:** any file using SDK hooks (`useFlowCurrentUser`,
  `useFlowQuery`, `useFlowMutate`, `useFlowConfig`) must start with `"use client"`.
- **Default network is Testnet.** To switch, edit the `config` object in
  `src/components/flow-provider-wrapper.tsx` — `README.md` documents exact Mainnet/Emulator
  blocks. `FlowProvider` receives `flowJson={flowJSON}`, so contract addresses resolve from
  `flow.json` aliases per network.
- **Address formatting:** user/recipient addresses are normalized to include a `0x` prefix
  before being passed to Cadence (see `flow-balance.tsx` and `send-flow.tsx`); `UFix64`
  amounts must be decimal strings — `send-flow.tsx` uses `parseFloat(amount).toFixed(8)`.
- **WalletConnect project ID in `flow-provider-wrapper.tsx` is an example** value
  (`9b70cfa398b2355a5eb9b1cf99f4a981`). Replace it before shipping.
- **Tailwind v4** is configured via `@tailwindcss/postcss` in `postcss.config.mjs` — there is
  no `tailwind.config.*` file; global styles live in `src/app/globals.css`.
